OPCIÓN 1: HERRAMIENTAS DEL NAVEGADOR (MÁS FÁCIL) ⭐
================================================================================

La forma MÁS RÁPIDA y FÁCIL es usar las herramientas de desarrollo del navegador:

GOOGLE CHROME / MICROSOFT EDGE:
--------------------------------

1. Abre tu sitio: http://127.0.0.1:8000/

2. Presiona F12 o Ctrl+Shift+I

3. Haz clic en el ícono de dispositivo móvil 📱 
   (Toggle device toolbar)
   O presiona: Ctrl+Shift+M

4. Selecciona el dispositivo que quieras simular:
   - iPhone 12 Pro
   - iPhone SE
   - Samsung Galaxy S20
   - iPad
   - Pixel 5
   - Y muchos más...

5. O crea dimensiones personalizadas:
   - 360 x 640 (común Android)
   - 375 x 667 (iPhone 8)
   - 414 x 896 (iPhone 11)

CARACTERÍSTICAS:
✓ Gratis y ya instalado
✓ No requiere instalación adicional
✓ Simula pantalla táctil
✓ Simula orientación (portrait/landscape)
✓ Simula throttling de red (3G, 4G)
✓ Permite captura de pantalla
✓ Inspecciona elementos en tiempo real

ATAJOS ÚTILES:
  Ctrl+Shift+M  → Toggle device toolbar
  Ctrl+Shift+C  → Inspect element
  F5            → Recargar
  Ctrl+R        → Recargar
  Ctrl+Shift+R  → Recargar sin caché

MOZILLA FIREFOX:
----------------

1. Abre tu sitio: http://127.0.0.1:8000/

2. Presiona F12

3. Haz clic en el ícono "Responsive Design Mode" 📱
   O presiona: Ctrl+Shift+M

4. Selecciona dispositivo o tamaño personalizado

CARACTERÍSTICAS:
✓ Similar a Chrome
✓ Simula touch events
✓ Captura de pantalla completa
✓ Modo responsive muy flexible

OPCIÓN 2: ANDROID STUDIO (EMULADOR COMPLETO) 🤖
================================================================================

Para una experiencia MÁS REAL, usa el emulador de Android Studio:

INSTALACIÓN:

1. Descarga Android Studio:
   https://developer.android.com/studio

2. Instala Android Studio (siguiente, siguiente...)

3. Abre Android Studio

4. Ve a: Tools → Device Manager

5. Crea un nuevo dispositivo virtual (AVD):
   - Clic en "Create Device"
   - Selecciona hardware (ej: Pixel 5)
   - Selecciona imagen del sistema (Android 11, 12, etc.)
   - Clic "Finish"

6. Inicia el emulador (botón ▶️)

USAR TU APP EN EL EMULADOR:

Opción A - Navegador del emulador:
  1. Abre Chrome en el emulador
  2. Navega a: http://10.0.2.2:8000/
     (10.0.2.2 es localhost desde el emulador)

Opción B - IP de tu PC:
  1. Averigua tu IP local: ipconfig
  2. En emulador navega a: http://TU_IP:8000/

VENTAJAS:
✓ Emulador REAL de Android
✓ Todas las funciones de Android
✓ Instalar apps reales
✓ GPS simulado
✓ Cámara simulada
✓ Sensores (acelerómetro, giroscopio)
✓ Diferentes versiones de Android

DESVENTAJAS:
✗ Requiere mucha RAM (mínimo 8GB)
✗ Descarga grande (~4GB)
✗ Más lento que herramientas del navegador
✗ Requiere virtualización habilitada en BIOS

REQUISITOS:
- Windows 10 64-bit o superior
- Mínimo 8GB RAM (recomendado 16GB)
- Virtualización habilitada (BIOS)
- Espacio en disco: ~10GB

OPCIÓN 5: USAR TU TELÉFONO REAL (MEJOR OPCIÓN) 📱
================================================================================

La MEJOR forma de probar es usar tu teléfono real:

PASOS:

1. CONECTAR A LA MISMA RED WIFI:
   - Tu PC y tu teléfono deben estar en la misma red WiFi

2. AVERIGUAR IP DE TU PC:
   Windows PowerShell:
   ```
   ipconfig
   ```
   
   Busca "IPv4 Address" de tu WiFi
   Ejemplo: 192.168.1.100

3. CONFIGURAR DJANGO PARA ACEPTAR CONEXIONES:
   
   En settings.py:
   ```python
   ALLOWED_HOSTS = ['localhost', '127.0.0.1', '192.168.1.100']
   # O más simple para desarrollo:
   ALLOWED_HOSTS = ['*']
   ```

4. REINICIAR SERVIDOR:
   ```
   python manage.py runserver 0.0.0.0:8000
   ```

5. EN TU TELÉFONO:
   Abre navegador y ve a:
   http://192.168.1.100:8000/
   (Usa tu IP real)

VENTAJAS:
✓ Prueba REAL con hardware real
✓ Performance real
✓ Touch real
✓ GPS real
✓ Sensores reales
✓ Sin emulación

DESVENTAJAS:
✗ Requiere misma red WiFi
✗ Requiere configurar ALLOWED_HOSTS

================================================================================
PROBLEMAS COMUNES Y SOLUCIONES
================================================================================

PROBLEMA: "Site can't be reached" desde teléfono
SOLUCIÓN:
  1. Verifica misma red WiFi
  2. Verifica ALLOWED_HOSTS en settings.py
  3. Verifica servidor corriendo: python manage.py runserver 0.0.0.0:8000
  4. Desactiva firewall temporalmente
  5. Verifica IP correcta con ipconfig

PROBLEMA: Funciona en PC pero no en emulador
SOLUCIÓN:
  1. Usa 10.0.2.2:8000 en lugar de localhost
  2. O usa tu IP local: 192.168.x.x:8000

PROBLEMA: Android Studio muy lento
SOLUCIÓN:
  1. Asigna más RAM al emulador
  2. Habilita aceleración hardware
  3. Usa imagen x86 en lugar de ARM
  4. Cierra otros programas

PROBLEMA: Responsive no se ve bien
SOLUCIÓN:
  1. Verifica viewport en HTML:
     <meta name="viewport" content="width=device-width, initial-scale=1.0">
  2. Usa unidades responsivas (%, rem, em)
  3. Usa Bootstrap grid system
  4. Media queries en CSS
